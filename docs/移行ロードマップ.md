# 住宅予測 → 社内規定LLM RAG 移行ロードマップ

最終更新: 2026-04-04

---

## Context

現行プロジェクトはCalifornia Housing予測のMLOpsパイプライン。これを完全に廃止し、`docs/仕様、設計書.md`に定義された**社内ドキュメント検索・QAシステム**（LLM RAG）に置き換える。GCPインフラ（Cloud Run, BigQuery, GCS, Elastic Cloud）とCI/CDパターンは流用し、ML固有コードは全て削除・置換する。

## 検証方法

### Phase 1 検証

```bash
cd terraform && terraform plan  # リソース差分確認
terraform apply                 # GCS/BQ/CloudRun/IAM作成
# BigQuery Vector Index作成（手動SQL）
```

### Phase 2 検証

```bash
# サンプルドキュメントをGCSにアップロード
gsutil cp data/sample/*.txt gs://doc-qa-bucket/

# Ingestion Job をローカル実行
cd src/doc-qa/ingestion && python main.py

# BigQuery確認
bq query 'SELECT doc_name, chunk_index, content FROM doc_qa_dataset.documents LIMIT 5'

# Elasticsearch確認（kuromoji検索）
curl -X GET "$ES_URL/doc-qa/_search" -d '{"query":{"match":{"content":"有給休暇"}}}'
```

### Phase 3 検証

```bash
# QA APIローカル起動
cd src/doc-qa/api && uvicorn main:app --reload

# ヘルスチェック
curl http://localhost:8000/health

# QAクエリ
curl -X POST http://localhost:8000/query -H 'Content-Type: application/json' \
  -d '{"query": "有給休暇の申請手続きを教えてください", "top_k": 5}'

# 回答に根拠ドキュメントと該当箇所が含まれることを確認
```

### Phase 4 検証

```bash
# テスト実行
cd src/doc-qa/ingestion && pytest -v
cd src/doc-qa/api && pytest -v

# Docker ビルド
make ingestion-build
make doc-qa-api-build

# CI/CD（GitHub Actionsでのテスト→デプロイ確認）
```
