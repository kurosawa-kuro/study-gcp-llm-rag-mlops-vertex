## Phase 7: RAG評価基盤 --- 実装済み

RAGシステムの品質を定量化する評価基盤を構築。検索品質と回答品質の2層で評価する。

```
scripts/eval/
├── evaluate.py          # メイン評価スクリプト（--search-type / --save-as / --top-k）
├── metrics.py           # 指標計算（Recall@K / MRR / Exact Match / ROUGE-L）
├── report.py            # 横並び比較レポート生成
├── queries_eval.jsonl   # 評価クエリ20件 + Ground Truth
└── results/             # 評価結果 JSON 保存先
```

### 評価指標

| 層 | 指標 | 説明 |
|---|------|------|
| 検索品質 | Recall@K (K=1,3,5,10) | 正解チャンクがTop-Kに含まれる割合 |
| 検索品質 | MRR | 最初の正解チャンクの順位の逆数の平均 |
| 回答品質 | Exact Match | 期待キーワードが全て含まれる割合 |
| 回答品質 | ROUGE-L | 文字レベルLCSベースの回答網羅性 |

### 検索パターン比較

`make eval-search-patterns` で vector / elasticsearch / hybrid の3パターンを比較し、ハイブリッド検索の効果を定量化する。
