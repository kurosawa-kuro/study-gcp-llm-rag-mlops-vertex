## Phase 1: インフラ構築 --- 実装済み

Terraform をモジュール分離構成で実装:

```
terraform/
├── provider.tf          # Google + Elastic Cloud（credentials.yml を yamldecode）
├── variables.tf         # ルート変数
├── main.tf              # モジュールオーケストレーション
├── outputs.tf           # API URL・ESエンドポイント・SA email
├── terraform.tfvars     # Elastic Cloud APIキー（git管理外）
└── modules/
    ├── data/            # GCS + BigQuery dataset/table
    ├── compute/         # Cloud Run Job/Service + Scheduler + Job invoker IAM
    ├── elastic/         # Elastic Cloud デプロイメント + Secret Manager
    ├── registry/        # Artifact Registry
    └── iam/             # SA + 最小権限バインディング
```

手動ステップ: BigQuery Vector Index 作成（`CREATE VECTOR INDEX ... OPTIONS (distance_type = 'COSINE')`）

## Phase 5: リファクタリング --- 実装済み

移行後に実施した構造改善:

| 施策 | 内容 |
|------|------|
| 設定一元化 | `shared/config.py` を唯一の設定ローダーに。全7箇所の YAML ロード重複を解消 |
| スクリプト基盤 | `shared/core.py` に gcloud / run / notify_discord / load_env を集約 |
| Terraform モジュール分離 | フラット10ファイル → 5モジュール（data/compute/elastic/registry/iam） |
| クレデンシャル統合 | `env/secret/credentials.yml` に GCP SA + Elastic Cloud を統合（YAML形式） |
| Dockerfile 修正 | build context をルートに変更。application.yml + config.py をコンテナに含む |
| シェルスクリプト廃止 | 全 .sh → Python 化（shared/core.py の共通関数で統一） |
| Makefile 統合 | `makefiles/*.mk` 4ファイル → `Makefile` 1ファイルに統合 |
| ES PoC 削除 | `src/elastic-search/` を削除（doc-qa に ES 統合済み） |
| サブフォルダ分離 | scripts/ → monitor/ops/setup、ingestion/ → extract/embed/store、api/ → endpoints/search/generation |

## 検証方法

```bash
# インフラ
make tf-plan && make tf-apply

# ドキュメント取込
python3 scripts/ops/upload_doc.py data/sample/
make ingestion-run
make ingestion-logs

# QA クエリ
make qa-api-url
python3 scripts/ops/query.py "有給休暇の申請手続きを教えてください"

# テスト
make test

# Docker ビルド
make ingestion-build
make qa-api-build

# 監視
make qa-api-monitor
python3 scripts/monitor/ingestion.py
```
