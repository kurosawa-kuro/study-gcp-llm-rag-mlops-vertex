## Phase 7: RAG評価基盤 --- 実装済み

RAGシステムの品質を定量化する評価基盤を構築。検索品質と回答品質の2層で評価する。

```
scripts/eval/
├── evaluate.py          # メイン評価スクリプト（--search-type / --save-as / --top-k）
├── metrics.py           # 指標計算（Recall@K / MRR / Exact Match / ROUGE-L）
├── report.py            # 横並び比較レポート生成
├── queries_eval.jsonl   # 評価クエリ20件 + Ground Truth
├── results/             # 評価結果 JSON 保存先
└── tests/
    ├── test_metrics.py  # metrics テスト
    └── test_report.py   # report テスト
```

### 評価指標

| 層 | 指標 | 説明 |
|---|------|------|
| 検索品質 | Recall@K (K=1,3,5,10) | 正解チャンクがTop-Kに含まれる割合 |
| 検索品質 | MRR | 最初の正解チャンクの順位の逆数の平均 |
| 回答品質 | Exact Match | 期待キーワードが全て含まれる割合 |
| 回答品質 | ROUGE-L | 文字レベルLCSベースの回答網羅性 |

### 検索パターン比較

`make eval-search-patterns` で vector / elasticsearch / hybrid の3パターンを比較し、ハイブリッド検索の効果を定量化する。

## Phase 8: Vertex AI Pipeline 統合 --- 実装済み

RAG評価基盤を Vertex AI Pipeline としてパッケージ化し、GCP 上で定期自動実行できるようにした。
Cloud Run Job 方式（直接実行）と Vertex AI Pipeline 方式（KFP オーケストレーション）が共存する。

### アーキテクチャ（デュアルパス）

```
=== パスA: Vertex AI Pipeline（KFP オーケストレーション）===

[CI/CD (GitHub Actions)]
  └── deploy-pipeline ジョブ
       ├── KFP JSON コンパイル
       ├── GCS にアップロード (gs://mlops-dev-a-doc-qa/pipeline-artifacts/)
       └── Vertex AI Pipeline スケジュール作成/更新（冪等）
             ↓
[Vertex AI Pipeline（毎週日曜 22:00 JST）]
  └── doc-qa-eval コンテナコンポーネント
       └── evaluate.py → 結果を GCS にアップロード

=== パスB: Cloud Run Job（直接実行・フォールバック）===

[Cloud Scheduler（毎週日曜 22:00 JST）]
  └── Cloud Run Job（doc-qa-eval）
       └── evaluate.py → 結果を GCS にアップロード
```

### 実装ファイル

```
src/doc-qa/pipeline/
├── pipeline.py              # KFP パイプライン定義（@dsl.pipeline）
├── run_pipeline.py          # compile / run / schedule コマンド（冪等スケジュール管理）
├── components/
│   └── run_evaluation.py    # コンテナコンポーネント（@dsl.container_component）
├── entrypoint.sh            # 環境変数設定 + evaluate.py 起動
├── Dockerfile               # 評価用イメージ（API モジュール + eval スクリプト同梱）
└── requirements.txt         # kfp>=2.7.0, google-cloud-aiplatform>=1.38.0
```

### Terraform リソース（compute モジュール）

| リソース | 説明 |
|---------|------|
| `google_cloud_run_v2_job.eval` | 評価用 Cloud Run Job（パスB） |
| `google_cloud_scheduler_job.eval_schedule` | Cloud Scheduler 週次トリガー（パスB） |
| ※ Vertex AI Pipeline スケジュール | CI/CD 管理（Terraform Provider 未対応のため） |

### CI/CD

`.github/workflows/doc-qa.yml` に3つのジョブ:

| ジョブ | 説明 |
|-------|------|
| `test` | ingestion / api のテスト実行 |
| `build-and-deploy` | 3コンポーネント（ingestion / api / pipeline）のビルド・プッシュ・デプロイ |
| `deploy-pipeline` | KFP コンパイル → GCS アップロード → Vertex AI Pipeline スケジュール作成/更新 |

### Makefile ターゲット

| コマンド | 説明 |
|---------|------|
| `make eval-pipeline-build` | 評価用 Docker イメージビルド |
| `make eval-pipeline-push` | イメージを Artifact Registry に push |
| `make eval-pipeline-compile` | パイプライン JSON コンパイル |
| `make eval-pipeline-run` | パイプラインをコンパイルして Vertex AI で実行 |
| `make eval-pipeline-schedule` | Vertex AI Pipeline 週次スケジュール作成/更新（冪等） |

### 技術選定: スケジュール管理

| 方式 | 状態 | 備考 |
|------|------|------|
| Terraform で管理 | 不可 | `google_vertex_ai_pipeline_schedule` リソースが Provider に存在しない |
| CI/CD で SDK 管理 | **採用** | `run_pipeline.py schedule` で冪等に作成/更新 |
