# 運用マニュアル

全操作は `make` コマンドで実行する。`make help` でコマンド一覧を表示。

---

## 1. 初回セットアップ

### 1-1. 前提条件

- Python 3.11+
- Docker
- GCP プロジェクト `mlops-dev-a` へのアクセス権

### 1-2. GCP CLI / Terraform インストール

```bash
python scripts/setup/gcp.py
python scripts/setup/terraform.py
gcloud init
```

### 1-3. GCP プロジェクト設定

```bash
make gcp-setup
```

内部で実行される処理:

| ステップ | 内容 |
|---------|------|
| `gcp-setup-apis` | 必要な GCP API を有効化（Cloud Run, BigQuery, Vertex AI 等） |
| `gcp-setup-sa` | Terraform SA に editor / run.admin 権限を付与 |
| `gcp-setup-vertex` | Compute SA に aiplatform.user 権限を付与 |
| `gcp-setup-docker` | Artifact Registry への Docker 認証設定 |

### 1-4. クレデンシャル配置

```bash
# GCP SA キー + Elastic Cloud APIキーを配置
# 構造は .env.example を参照
vi env/secret/credentials.yml

# Terraform 変数ファイルを配置
vi terraform/terraform.tfvars
```

### 1-5. インフラ構築

```bash
make tf-plan     # 差分確認
make tf-apply    # 全リソース作成
```

作成されるリソース:

| モジュール | リソース |
|-----------|---------|
| `data` | GCS バケット, BigQuery dataset + table |
| `compute` | Cloud Run Job (Ingestion), Cloud Run Service (API), Cloud Scheduler |
| `elastic` | Elastic Cloud デプロイメント, Secret Manager |
| `registry` | Artifact Registry |
| `iam` | Service Account + 最小権限 IAM |

### 1-6. BigQuery Vector Index 作成（手動）

```sql
CREATE VECTOR INDEX embedding_index
ON doc_qa_dataset.documents(embedding)
OPTIONS (distance_type = 'COSINE');
```

## 2. 日常運用

### 2-1. ドキュメント取込

```bash
# GCS にドキュメントをアップロード
python scripts/ops/upload_doc.py data/sample/

# Ingestion Job を実行
make ingestion-run

# 実行履歴を確認
make ingestion-logs
```

Cloud Scheduler により毎日 9:00 JST に自動実行される。

### 2-2. QA クエリ

```bash
# API の URL を確認
make qa-api-url

# クエリを送信
python scripts/ops/query.py "有給休暇の申請手続きを教えてください"
python scripts/ops/query.py "出張の宿泊費上限は？" 3
```

### 2-3. 監視

```bash
# QA API の健全性チェック（/health + Discord 通知）
make qa-api-monitor

# Ingestion Job の実行結果確認（+ Discord 通知）
python scripts/monitor/ingestion.py
```

Discord 通知を有効にするには `.env` に `DISCORD_WEBHOOK_URL` を設定する。

### 2-4. ログ確認

```bash
make qa-api-logs       # QA API のログ
make ingestion-logs    # Ingestion Job の実行履歴
```

## 3. デプロイ

### 3-1. 全体デプロイ

```bash
make deploy
```

内部フロー: `build → push → Cloud Run 更新`（Ingestion + QA API 両方）

### 3-2. 個別デプロイ

```bash
make ingestion-deploy    # Ingestion のみ
make qa-api-deploy       # QA API のみ
```

### 3-3. テストのみ

```bash
make test              # 全テスト
make ingestion-test    # Ingestion のみ
make qa-api-test       # QA API のみ
```

### 3-4. Docker ビルドのみ（push なし）

```bash
make ingestion-build
make qa-api-build
```

## 5. 設定変更

### 5-1. プロジェクト設定

```
env/config/application.yml    ← 唯一の定義元
```

変更後は再デプロイが必要:

```bash
make deploy
```

主要な設定項目:

| キー | 説明 | デフォルト |
|-----|------|-----------|
| `gcp.project_id` | GCP プロジェクト | mlops-dev-a |
| `gcp.region` | リージョン | asia-northeast1 |
| `storage.bucket_name` | GCS バケット | mlops-dev-a-doc-qa |
| `bigquery.dataset` | BQ データセット | doc_qa_dataset |
| `elasticsearch.index_name` | ES インデックス | doc-qa |
| `ingestion.chunk_size` | チャンクサイズ（文字） | 800 |
| `ingestion.chunk_overlap` | オーバーラップ（文字） | 50 |
| `ingestion.embedding_model` | Embedding モデル | text-multilingual-embedding-002 |
| `api.gemini_model` | Gemini モデル | gemini-2.0-flash |
| `api.default_top_k` | 検索上位件数 | 5 |
| `api.rrf_k` | RRF 定数 | 60 |

### 5-2. クレデンシャル

| ファイル | 内容 | git管理 |
|---------|------|---------|
| `env/secret/credentials.yml` | GCP SA + Elastic Cloud | 管理外 |
| `terraform/terraform.tfvars` | Terraform 変数 | 管理外 |

## 6. トラブルシューティング

### Ingestion Job が失敗する

```bash
make ingestion-logs    # 直近5件の実行履歴

# Cloud Logging で詳細ログ
gcloud logging read \
  'resource.type=cloud_run_job AND resource.labels.job_name=doc-qa-ingestion' \
  --limit=50 --format=json --project=mlops-dev-a
```

### QA API が応答しない

```bash
make qa-api-url        # URL 確認
make qa-api-monitor    # /health チェック
make qa-api-logs       # 直近ログ確認
```

### Terraform apply が失敗する

```bash
make tf-validate       # 構文チェック
make tf-plan           # エラー詳細確認
make tf-fmt            # フォーマット修正
```

### Docker ビルドが失敗する

build context はプロジェクトルート:

```bash
docker build -f src/doc-qa/api/Dockerfile -t test .
docker build -f src/doc-qa/ingestion/Dockerfile -t test .
```
