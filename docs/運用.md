# 運用マニュアル

全操作は `make` コマンドで実行する。`make help` でコマンド一覧を表示。

---

## 1. 初回セットアップ

### 1-1. 前提条件

- Python 3.11+
- Docker
- GCP プロジェクト `mlops-dev-a` へのアクセス権
- `gcloud` / `terraform` インストール済み（未インストール時は下記参照）

```bash
python3 scripts/setup/gcp.py       # 未インストール時のみ実行（自動スキップ）
python3 scripts/setup/terraform.py  # 未インストール時のみ実行（自動スキップ）
```

### 1-2. GCP ログイン確認

```bash
gcloud config list    # 設定済みか確認
# 未設定の場合のみ: gcloud init
```

### 1-3. クレデンシャル配置

```bash
# GCP SA キーを配置（構造は .env.example 参照）
vi env/secret/credentials.yml

# Elastic Cloud Org APIキーを配置
vi terraform/terraform.tfvars
```

`terraform.tfvars` に必要な変数:

```hcl
elastic_cloud_api_key = "<Elastic Cloud Org APIキー>"
```

ES接続用の username/password は `ec_deployment` から自動取得されるため手動設定不要。

### 1-4. 全環境構築（冪等）

```bash
make deploy-all
```

これ1コマンドで以下が全て実行される:

| 順序 | ステップ | 内容 |
|------|---------|------|
| 1 | `gcp-setup` | API 有効化 + SA 権限付与 + Docker 認証 |
| 2 | `tf-bootstrap` | Terraform init → Registry import(冪等) → Registry 作成 |
| 3 | `ingestion-push` | Ingestion イメージ build → push |
| 4 | `qa-api-push` | QA API イメージ build → push |
| 5 | `tf-apply` | 全 GCP リソース作成 |

GCP SA に付与される権限:

| ロール | 用途 |
|--------|------|
| `roles/editor` | 全般的なリソース管理 |
| `roles/run.admin` | Cloud Run 管理 |
| `roles/resourcemanager.projectIamAdmin` | IAM ポリシー管理 |
| `roles/secretmanager.admin` | Secret Manager 管理 |

### 1-5. BigQuery Vector Index 作成（手動・初回のみ）

```sql
CREATE VECTOR INDEX embedding_index
ON doc_qa_dataset.documents(embedding)
OPTIONS (distance_type = 'COSINE');
```

## 2. 日常運用

### 2-1. ドキュメント取込

```bash
python3 scripts/ops/upload_doc.py data/sample/   # GCS にアップロード
make ingestion-run                                # Ingestion Job 実行
make ingestion-logs                               # 実行履歴確認
```

Cloud Scheduler により毎日 9:00 JST に自動実行される。

### 2-2. QA クエリ

```bash
make qa-api-url                                                   # API URL 確認
python3 scripts/ops/query.py "有給休暇の申請手続きを教えてください"     # クエリ送信
python3 scripts/ops/query.py "出張の宿泊費上限は？" 3                  # top_k 指定
```

### 2-3. 監視

```bash
make qa-api-monitor                    # QA API /health チェック + Discord 通知
python3 scripts/monitor/ingestion.py   # Ingestion Job 結果確認 + Discord 通知
```

Discord 通知を有効にするには `.env` に `DISCORD_WEBHOOK_URL` を設定。

### 2-4. ログ確認

```bash
make qa-api-logs       # QA API のログ
make ingestion-logs    # Ingestion Job の実行履歴
```

## 3. デプロイ

### 3-1. 全構築 / 全削除

```bash
make deploy-all     # 全環境構築（冪等・何度実行しても安全）
make destroy-all    # 全 GCP リソース削除（課金停止）
```

### 3-2. アプリのみ再デプロイ

```bash
make deploy              # Ingestion + QA API 両方
make ingestion-deploy    # Ingestion のみ
make qa-api-deploy       # QA API のみ
```

### 3-3. テスト

```bash
make test              # 全テスト
make ingestion-test    # Ingestion のみ
make qa-api-test       # QA API のみ
```

### 3-4. Docker ビルドのみ（push なし）

```bash
make ingestion-build
make qa-api-build
```

## 5. 設定変更

### 5-1. プロジェクト設定

```
env/config/application.yml    ← 全設定の唯一の定義元
```

変更後は再デプロイが必要:

```bash
make deploy
```

主要な設定項目:

| キー | 説明 | デフォルト |
|-----|------|-----------|
| `gcp.project_id` | GCP プロジェクト | mlops-dev-a |
| `gcp.region` | リージョン | asia-northeast1 |
| `storage.bucket_name` | GCS バケット | mlops-dev-a-doc-qa |
| `bigquery.dataset` | BQ データセット | doc_qa_dataset |
| `elasticsearch.index_name` | ES インデックス | doc-qa |
| `ingestion.chunk_size` | チャンクサイズ（文字） | 800 |
| `ingestion.chunk_overlap` | オーバーラップ（文字） | 50 |
| `ingestion.embedding_model` | Embedding モデル | text-multilingual-embedding-002 |
| `api.gemini_model` | Gemini モデル | gemini-2.0-flash |
| `api.default_top_k` | 検索上位件数 | 5 |
| `api.rrf_k` | RRF 定数 | 60 |

### 5-2. クレデンシャル

| ファイル | 内容 | git管理 |
|---------|------|---------|
| `env/secret/credentials.yml` | GCP SA キー + Elastic Cloud Org APIキー | 管理外 |
| `terraform/terraform.tfvars` | `elastic_cloud_api_key`（ec プロバイダ認証用） | 管理外 |

ES 接続用の `username` / `password` は Terraform が `ec_deployment` から自動取得し Secret Manager に格納する。手動管理不要。

## 6. トラブルシューティング

### Ingestion Job が失敗する

```bash
make ingestion-logs    # 直近5件の実行履歴

# Cloud Logging で詳細ログ
gcloud logging read \
  'resource.type=cloud_run_job AND resource.labels.job_name=doc-qa-ingestion' \
  --limit=50 --format=json --project=mlops-dev-a
```

### QA API が応答しない

```bash
make qa-api-url        # URL 確認
make qa-api-monitor    # /health チェック
make qa-api-logs       # 直近ログ確認
```

### Terraform apply が失敗する

```bash
make tf-validate       # 構文チェック
make tf-plan           # エラー詳細確認
make tf-fmt            # フォーマット修正
```

### Docker イメージが更新されない

キャッシュが効いて変更が反映されない場合:

```bash
docker build --no-cache -f src/doc-qa/api/Dockerfile -t TAG .
docker build --no-cache -f src/doc-qa/ingestion/Dockerfile -t TAG .
```
